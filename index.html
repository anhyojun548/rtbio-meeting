왜 필요한가

수기 업무를 자동화

하루 30건 발주를 엑셀·수기로 처리하면서 오류·중복·마감 지연이 반복적으로 발생합니다.

무엇을 만드는가

맞춤형 커스텀 ERP

발주 → 거래명세서 → 정산 → 마감 보고서까지 한 시스템에서 자동으로 처리되는 RTBIO 전용 시스템.

PROJECT OVERVIEW

1. 프로젝트 개요

현재 상황

RTBIO는 발주 접수, 재고 관리, 정산, 영업 데이터 관리 등 핵심 업무를 엑셀·수기 중심으로 처리하고 있습니다. 이로 인해 다음과 같은 문제가 반복됩니다.

📋

같은 내용을 여러 번 입력

발주서, 거래명세서, 정산표에 같은 데이터를 반복 입력

⚠️

숫자 실수와 누락

수기 입력 과정에서 금액 오류, 수량 착오 발생

📊

실시간 현황 파악 불가

미수금이 얼마인지, 어떤 발주가 처리됐는지 즉시 확인 불가

🕐

월말 마감에 시간 소모

각종 자료를 모아 수작업으로 마감 보고서 제작

목표

  • 경영지원팀 수기 업무(하루 약 30건 발주 처리)를 시스템 기반으로 자동화
  • 부서별 데이터를 하나의 통합 시스템으로 관리
  • 향후 앱 확장, 의료기기 추적, 생산 연동까지 단계적으로 확장 가능한 구조 구축

기존 ERP와의 차별점

본 시스템은 RTBIO의 업무 흐름에 100% 맞춤 설계된 커스텀 ERP입니다.

항목기존 ERP (더존, SAP 등)본 시스템
커스터마이징어렵고 비쌈RTBIO 업무에 100% 맞춤

확장모듈 구매 필요필요 기능 직접 추가
비용 구조라이선스 비용 지속 발생유지보수 계약으로 전환

5. 단계별 기능 로드맵

한 번에 모든 걸 만들지 않습니다. 가장 급한 것부터 단계적으로 확장합니다.

📌 현장 프로세스 파악 후 최종 확정
아래 기능 목록은 현재까지 파악된 정보를 바탕으로 한 초안입니다. 계약 확정 후 하마다랩스 PM이 RTBIO 현장에 상주하여 각 부서 실무자와 실제 업무 프로세스를 직접 파악하고, 이 결과를 반영해 기능을 재정의·추가·조정합니다.

Phase 1 — 경영지원팀 자동화 1차 범위

하루 30건 수기 처리 → 클릭 기반 자동 처리. 메인 시스템 + DB 인프라 구축 포함.

Phase 2 — 품질관리팀 재고 자동화 2차 범위

발주 확정 시 재고 자동 차감, 입출고 관리, 부족 시 경고, 실시간 재고 대시보드.

Phase 3 — 영업부 대시보드 3차 범위

영업사원이 거래처별 매출·미수금·실적을 직접 조회하는 권한 분리형 대시보드.

Phase 4 — 베트남 생산·해외 재고 연동 별도 프로젝트

국내·해외 재고 통합 조회, 부족 예측 알림. 현지 환경 파악이 필요해 별도 견적으로 진행합니다.

8. 가격 정책 및 계약 구조

단계별 가격은 상단 요약 섹션을 참고해 주세요. 여기서는 가격 산정 근거와 계약 방식을 설명합니다.

1차가 가장 큰 폭으로 할인되는 이유
1차에는 메인 시스템과 데이터베이스 등 앞으로 2, 3차에서 계속 사용할 인프라가 통째로 포함됩니다. 처음 시작하시는 단계의 부담을 최소화하기 위해 가장 큰 폭의 할인을 적용했습니다. 2~3차는 1차 인프라(DB, 인증, UI)를 그대로 재사용하기 때문에 단독 진행 대비 단가가 자연스럽게 절감됩니다.
